### [C2-01] (Critical) config.yaml の予算費目構造が様式2-1テーブルの行構成と不整合 → **対応済み**

- **箇所**: docs/prompts.md Prompt 1-1（config.yaml budget設計）:212-224行
- **前回対応状況**: 新規
- **対応内容**: config.yaml の budget.yearly を equipment/consumables/travel/personnel/other の5費目に分離。budget.details（品目レベル積算）を追加。様式2-1の記載注意事項も追記。
- **内容**: prompts.md の config.yaml 予算設計では `items: 物品費（設備備品費 + 消耗品費）` を単一フィールドとして定義している。しかし様式2-1のTable 3（9行×7列）は以下の行構成を持つ:

  | 行 | 費目 |
  |---|------|
  | 0 | ヘッダ（年度列） |
  | 1 | 設備備品費 |
  | 2 | 消耗品費 |
  | 3 | （小計: 物品費） |
  | 4 | 旅費 |
  | 5 | 人件費・謝金 |
  | 6 | その他 |
  | 7 | 直接経費合計 |
  | 8 | 間接経費 |

  config.yaml の `items` を1つのフィールドで持つと、設備備品費と消耗品費を個別の行に書き込めない。別紙3「研究費の取扱区分表」でも設備備品費（耐用年数1年以上 かつ 取得価格10万円以上）と消耗品費は明確に区分されている。

- **影響**: fill_forms.py が様式2-1テーブルの正しい行にデータを書き込めない。実装開始後にYAMLスキーマの再設計が必要となり、全後続スクリプト（fill_forms.py, fill_excel.py）に波及する。
- **推奨対応**: config.yaml の budget.yearly を以下の構造に変更:
  ```yaml
  budget:
    yearly:
      - year: 1
        equipment: 0       # 設備備品費（千円）
        consumables: 0     # 消耗品費（千円）
        travel: 0          # 旅費（千円）
        personnel: 0       # 人件費・謝金（千円）
        other: 0           # その他（千円）
        # 物品費 = equipment + consumables（自動計算）
        # 間接経費 = 直接経費合計 × indirect_rate（自動計算）
  ```

### [C2-02] (Critical) 様式2-2の詳細積算データ構造が未設計 → **対応済み**

- **箇所**: docs/prompts.md Prompt 1-1, Prompt 3-1
- **前回対応状況**: 新規
- **対応内容**: config.yaml に budget.details セクション（year/line_items/category/name/quantity/unit_price/amount/justification）を追加。
- **内容**: 様式2-2（Tables 5-9、各12行×5列）は**品目レベルの積算明細表**である。各行に以下を記載する:

  | 列 | 内容 |
  |----|------|
  | 0 | 項目（メーカー名、品名、仕様等） |
  | 1 | 数量 |
  | 2 | 単価 |
  | 3 | 金額 |
  | 4 | 使用目的及び必要性 |

  別紙2 §4【様式2-2】には「各年度の経費について、積算内訳を記載してください。使用目的及び必要性の欄には、当該経費の計上が研究計画上必要である理由を簡潔に記載してください。」とある。

  しかし config.yaml の budget 設計にはカテゴリ合計額しかなく、個別品目（装置名、試薬名、旅費の出張先等）のデータ構造が存在しない。fill_forms.py は品目レベルでセルに書き込む必要があるが、データソースがない。

- **影響**: 様式2-2は全タイプで必須（○）。fill_forms.py 実装時に品目リストのデータ構造を新規設計する必要があり、config.yaml のスキーマに大規模な追加が発生する。
- **推奨対応**: config.yaml に `budget.details` セクションを追加するか、別ファイル `budget_details.yaml` を新設:
  ```yaml
  budget_details:
    - year: 1
      items:
        - category: equipment  # or consumables, travel, personnel, other
          name: "○○測定装置（メーカー名、型番）"
          quantity: 1
          unit_price: 5000  # 千円
          amount: 5000
          justification: "○○の測定に必要"
  ```

## パイプライン実行可能性ドライラン結果

### Step 0 (Docker): 実現可能 ✓ ただし未着手

- python:3.12-slim + Pandoc 3.6.x (deb) + pip packages + fonts-noto-cjk の構成は実現可能
- fonts-noto-cjk は ~150MB でDockerイメージが肥大化するが、日本語CJK処理に必須
- docker-compose.yml のボリュームマウント: `../../:/workspace`（docker/ からの相対パス）で問題なし

### Step 1 (YAML): 設計に重大な修正が必要 ✗

- **config.yaml**: 予算費目構造の再設計が必要（C2-01, C2-02）
- **researchers.yaml**: 論文5本制限、全所属機関の強調が必要（C2-05）
- **other_funding.yaml**: 秘密保持契約フラグの追加を検討（C2-16）
- **security.yaml**: 設計は概ね妥当。別紙5の24テーブルと別添の13テーブルの対応は、別添(tables 0-12) が別紙5(tables 3-15) のサブセットであり、データ共通化の設計は正しい

### Step 2 (Markdown): 見出し修正が必要 ✗

- **様式1-2**: セクション見出し5箇所の修正が必要（C2-04）
- **概要図**: Pandocで画像をdocxに埋め込む際、PNGで300dpi→概要図1枚で0.5-1.5MB。3MB目標との兼ね合いでは、150dpiへの圧縮またはSVG→EMF変換を検討
- **east_asian_line_breaks**: Pandoc 3.6.x でサポート確認済み。日本語テキストの改行がスペースにならない

### Step 3 (Word): 複数の技術的課題あり ⚠

- **fill_forms.py**: セルマッピング表は今回の解析で概ね特定済み（C2-14）。cell.paragraphs[0].runs 操作によるフォント維持は prompts.md に記載済みで正しい方針
- **行の動的追加**: 様式3-1/3-2で他制度エントリが複数ある場合のテーブル行追加は python-docx で可能だが、罫線スタイルの手動設定が必要
- **build_narrative.sh**: reference-doc の作成手順が未定義（C2-09）。カスタムテンプレート作成が前提
- **様式削除**: 不要様式の段落・テーブル削除ロジックが必要（C2-08）

### Step 4 (Excel): openpyxl制約に要注意 ⚠

- **様式6**: 「リスト」シートに研究テーマ23種（+フィールド名等）のドロップダウンソースがある。config.yaml の theme_number → テーマ名文字列への変換が必要。ドロップダウン値は「リスト」シートの内容と**完全一致**させること
- **Data Validation**: xlsx14 拡張が openpyxl 非対応（C2-12）。保存後の検証が必須
- **条件付き書式**: 19ルール存在（C2-13）。破損リスクあり
