- **箇所**: main/ 以下の全ディレクトリ
- **内容**: 全ドキュメント（CLAUDE.md, SPEC.md, README.md, prompts.md）は以下のディレクトリ名を参照している:

### [M-02] (Major) openpyxlが様式6のデータバリデーション拡張に非対応

- **箇所**: data/source/r08youshiki6.xlsx
- **内容**: openpyxl でこのファイルを読み込むと以下の警告が発生:
  ```
  UserWarning: Data Validation extension is not supported and will be removed
  ```
  これは xlsx14 拡張形式のデータバリデーション（ドロップダウンリスト等）が openpyxl で非対応であり、**保存時に削除される**ことを意味する。様式6のドロップダウン（研究テーマ23種、分野、タイプS/A/C等の選択リスト）が失われる。

- **推奨対応**: (1) openpyxl で保存後のファイルをExcelで開いてドロップダウンが維持されているか確認する。(2) 消失する場合、xlsx を ZIP として展開し、xl/worksheets/sheet1.xml の extLst 要素を手動で保持する後処理を実装する。(3) 代替として xlwings（要Windows）や xlsxwriter の使用を検討。

### [M-04] (Major) 様式1-1テーブルの大規模セル結合がpython-docx操作の障壁

- **箇所**: data/source/r08youshiki1_5.docx Table 0
- **内容**: 様式1-1（申請概要テーブル、Table 0）は20行×8列だが、**105箇所以上のセル結合**が存在する。これは全160セルの65%超が結合されていることを意味し、実質的なレイアウトは単純な行列構造ではない。python-docx で結合セルにアクセスする場合、`row.cells` が重複したセルオブジェクトを返すため、単純な行列インデックスでの書き込みが正しく動作しない。

  他のテーブルでも同様:

  - Table 12, 13（様式4-1/4-2: CV）: 10行×5列中 ~21箇所結合
- **影響**: テーブル操作コードが意図しないセルに書き込むバグが頻発する。prompts.md Prompt 3-1 で「セル内テキストのパターンマッチで特定」と記載されているが、結合セルのテキスト取得も同様に注意が必要。
- **推奨対応**: fill_forms.py 実装時に、各テーブルの結合構造を事前調査するユーティリティ関数を作成する。`cell.text` でのパターンマッチに加え、XML直接操作（lxml 経由で `w:tc` 要素を特定）のフォールバックを検討する。

### [M-05] (Major) 様式1-2（Pandoc生成docx）と様式1-5（python-docx記入済みdocx）の統合方法が未定義

- **箇所**: SPEC.md §3.1, prompts.md Prompt 3-2
- **内容**: SPEC.md §3.1 の出力仕様では `youshiki1_2_narrative.docx` を「様式1-5.docxに組み込み or 単独PDF」と記載しているが、統合方法が確定していない。prompts.md Prompt 3-2 では「最終的にはこのdocxの内容を様式1-5のdocxに統合するか、またはWindows側で手動コピー＆ペースト」と曖昧に記載。

  問題は、e-Rad への提出が「様式1-1〜5 + 参考様式 を**1つのPDF**に結合」であること（募集要項 §4.4.2）。Pandoc生成のdocxとpython-docx記入済みのdocxを1つのWordファイルに統合する手法が定義されておらず、python-docxでの文書結合は段落・セクションの書式が崩れる高リスク操作である。
- **影響**: 最終提出物の生成がブロックされる。Windows側での手動コピペに頼る場合、再現可能ビルドの設計方針に反する。
- **推奨対応**: 以下のいずれかを選択してSPEC.mdに明記する:
  1. **Windows側でPDF結合**: 各docxを個別にPDF化した後、PDF結合ツール（pypdfまたはAcrobat）で1つにまとめる
  2. **python-docxで文書結合**: Pandoc出力のdocxからparagraphsとtablesを抽出し、様式1-5のdocxの適切な位置に挿入する（OOXML直接操作が必要）
  3. **Pandocのみで対応**: 様式1-2, 1-3の内容をPandocで生成し、テーブルフォーム部分もPandocの変数機能で記入する（大規模な設計変更）
