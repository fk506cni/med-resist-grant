# 共同執筆 使い方ガイド

このフォルダは、申請書（令和8年度 安全保障技術研究推進制度）を共同で執筆するための共有フォルダです。
ここにある Markdown ファイルを編集すると、自動的に Word/PDF に変換されます。

---

## フォルダ構成

```
med-resist-collab/
├── README_使い方.md   ← このファイル
├── drafts/             ← 原稿ファイル（ここを編集します）
│   ├── youshiki1_2.md  … 様式1-2: 研究計画の詳細（メイン原稿）
│   ├── youshiki1_3.md  … 様式1-3: 追加説明事項
│   └── figs/           … 図・画像ファイル置き場
├── trigger.txt         ← ビルド実行の指示ファイル
├── status.txt          ← 現在の処理状況
└── products/           ← 完成した PDF・Word・Excel ファイル
```

---

## 1. 原稿の編集方法

### 編集するファイル

`drafts/` フォルダ内の `.md` ファイルを編集してください。

- **youshiki1_2.md** — 研究計画の詳細（最大15ページ。メインの執筆箇所です）
- **youshiki1_3.md** — 追加説明事項

### エディタの選び方

お好みの方法で編集できます。

#### 方法A: Google Drive 上で直接編集（最も簡単）

1. ファイルをダブルクリックして開く
2. 「テキストエディタ」で開かれます
3. そのまま編集・保存

※ Markdown のプレビュー（装飾表示）はできませんが、手軽に編集できます。

#### 方法B: ブラウザ版 VS Code で編集（プレビュー付き）

1. ブラウザで https://vscode.dev を開く
2. `.md` ファイルをブラウザにドラッグ＆ドロップ
3. 右上の「プレビューを横に表示」ボタンで、仕上がりイメージを確認しながら編集
4. 編集後、ファイルを保存してダウンロード
5. Google Drive の `drafts/` フォルダにアップロード（上書き）

#### 方法C: お手元のパソコンで編集

1. ファイルをダウンロード
2. お好きなテキストエディタ（メモ帳、サクラエディタ、VS Code など）で編集
3. 保存後、Google Drive の `drafts/` フォルダにアップロード（上書き）

### 図・画像の追加方法

1. 画像ファイル（PNG、JPEG など）を `drafts/figs/` フォルダにアップロード
2. 原稿の挿入したい位置に、以下のように記述

```
![図の説明文](figs/ファイル名.png)
```

例:

```
![図1 耐性菌の分離頻度](figs/fig1_resistance_rate.png)
```

## 2. ビルド（原稿 → PDF 変換）の実行方法

原稿の編集が終わったら、以下の手順で PDF を生成できます。

### 手順

1. 共有フォルダ内の **`trigger.txt`** を開く
2. 中身を **`build`** に書き換えて保存する
3. **Google Chat に通知が届きます**（各工程の進捗がメッセージで届きます）
4. 全工程が完了すると「🎉 全工程完了」の通知が届きます
5. **`products/`** フォルダに PDF 等が配信されます

### 処理の流れ（自動で実行されます）

```
trigger.txt に "build" と書いて保存
  ↓
📥 原稿ファイルの同期
  ↓
🔨 Word / Excel ファイルの生成
  ↓
📤 PDF への変換
  ↓
📦 products/ フォルダに成果物を配信
  ↓
🎉 完了（Google Chat に通知）
```

### 処理状況の確認

**`status.txt`** を開くと、現在の状態が表示されています。

| 表示 | 意味 |
|------|------|
| IDLE | 待機中（ビルド可能） |
| 🔄 ビルドパイプライン実行中... | 処理中 |
| ✅ 完了 (日時) | 正常に完了 |
| ❌ エラー: ... | エラーが発生 |

---

## 3. 成果物の確認

1. ビルド完了後、**`products/`** フォルダを開く
2. PDF ファイルを開いて、レイアウト・内容を確認
3. 修正が必要であれば `drafts/` のファイルを編集して、再度ビルドを実行

---

## 4. 注意事項

- **ビルド中は原稿を編集しないでください**
  `status.txt` が「ビルド中」の間は、`drafts/` 内のファイルを変更しないでください。変更が反映されないことがあります。

- **連続ビルドは2分間の間隔が必要です**
  前回のビルド完了から2分以内に再度 trigger.txt に "build" と書いても、スキップされます。少し待ってから再実行してください。

- **Google Drive の同期には数秒〜十数秒かかります**
  ファイルを保存してからビルドを実行する場合は、少し間を空けてください。

- **問題が起きた場合**
  Google Chat でご連絡ください。エラー内容は status.txt や Google Chat の通知で確認できます。

## 5. Markdown 記法クイックリファレンス

申請書の執筆で使う主な書き方の一覧です。

### 見出し

```markdown
# 大見出し（様式のセクション）
## 中見出し
### 小見出し
```

### 箇条書き

```markdown
- 項目1
- 項目2
  - 入れ子の項目
```

番号付き:

```markdown
1. 最初の項目
2. 次の項目
3. さらに次の項目
```

### 強調

```markdown
**太字にしたいテキスト**
```

### 表

```markdown
| 項目 | 内容 | 備考 |
|------|------|------|
| A    | 説明 | ここに補足 |
| B    | 説明 | ここに補足 |
```

### 図の挿入

```markdown
![図の説明文](figs/ファイル名.png)
```

### 改ページ

```markdown
\newpage
```

### コメント（PDF には表示されません）

原稿内にメモを残したい場合、以下の書き方で非表示のコメントを書けます。

```markdown
<!-- ここにコメントを書く。PDF出力には含まれません -->
```

例:

```markdown
<!-- TODO: この段落のデータを最新版に更新する -->
```

## 困ったときは

| 状況 | 対処 |
|------|------|
| ビルドが始まらない | trigger.txt の中身が正確に `build` になっているか確認してください |
| エラーが出た | status.txt と Google Chat の通知を確認し、ご連絡ください |
| PDF のレイアウトが崩れた | Markdown の書き方に問題がある可能性があります。ご連絡ください |
| 画像が表示されない | ファイル名（大文字・小文字・拡張子）が原稿の記述と一致しているか確認してください |
